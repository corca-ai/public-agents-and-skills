# AI 네이티브 제품 팀 문서 윤문 요구사항

## Original Requirement
"@AI_NATIVE_PRODUCT_TEAM.md 를 면밀히 읽고, 내 의도를 파악하고, 윤문하라."

## Decisions Made

| 질문 | 결정 |
|------|------|
| 대상 독자 | 내부 팀원 + 외부 공개용 |
| 윤문 방향 | 가독성 + 구체화 + 문체 통일 |
| 구조 변경 | 적극적 재구성 가능 |
| 어조 | 선언적/비전적 (매니페스토 스타일) |
| 기술 용어 | 그대로 유지 |
| 그룹화 | 3개 섹션 (AI 협업 / 조직 문화 / 기술 실천) |

---

## Before (Original)

8개 항목 나열식 구조:
1. 코드리뷰 → 프롬프트 리뷰/짝 프롬프팅
2. 프롬프트 자산화, 실험/공유 문화
3. 직군 역할 모호화
4. 모든 일의 코드/LLM 대체 믿음
5. 코드 품질 투자
6. papercut work 실현
7. 새 프로젝트는 한 사람이 E2E
8. 자료조사→스펙→정교화→학습→구현→회고→코드리뷰 싸이클 (7개 하위 항목 포함)

**문제점**:
- 항목 간 관계가 불명확
- 문체 혼용 (~한다, ~된다, ~늘어난다 등)
- 긴 문장으로 가독성 저하
- 제목/소제목 부재

---

## After (Clarified)

3개 섹션 그룹화 구조:

### 1. AI와 협업하는 방식
- 프롬프트 리뷰와 짝 프롬프팅
- 프롬프트의 자산화
- 스펙-구현-회고 싸이클 (7단계)

### 2. 조직 문화의 변화
- 직군의 경계가 흐려진다
- 모든 일은 자동화할 수 있다
- 한 사람이 처음부터 끝까지

### 3. 기술적 실천
- 코드 품질에 투자한다
- Papercut을 해결한다

**개선 사항**:
- 논리적 그룹화로 구조 명확화
- 선언적 어조로 문체 통일
- 짧은 문장으로 가독성 향상
- 제목과 소제목 추가

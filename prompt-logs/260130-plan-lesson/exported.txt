     Plan: Add Plan & Lessons Protocol

     Goal

     Add a protocol to CLAUDE.md that ensures every plan-mode session
     produces persistent artifacts:
     - plan.md with success criteria (BDD-style acceptance tests) and
     deferred actions
     - lessons.md with learnings accumulated throughout the session
     (ping-pong + implementation)

     Files to Create/Modify

     1. Create docs/plan-and-lessons.md — detailed protocol
     2. Edit CLAUDE.md — add link to the new doc

     docs/plan-and-lessons.md Content

     Plan Document

     - Location: prompt-logs/{YYMMDD}-{title}/plan.md, path determined by
     Claude based on user's request
     - Must include:
       - Success Criteria: BDD-style acceptance tests (Given/When/Then)
       - Deferred Actions: requests received during plan mode that can't be
     handled immediately, tracked as checklist
     - Timing: created when entering plan mode, before implementation begins

     Lessons Document

     - Location: prompt-logs/{YYMMDD}-{title}/lessons.md, same directory as
     plan
     - Records:
       - Learnings from user ping-pong (clarifications, corrected
     assumptions, revealed preferences)
       - Implementation learnings (expected vs actual, gaps, takeaways)
     - Timing: accumulated incrementally throughout the session, not just at
      the end
     - Format: follows the Expected/Actual/Takeaway structure from
     plan-mode.md, with optional When [situation] → [action] guidelines

     Deferred Actions

     - During plan mode, only plan file can be edited
     - User requests that can't be handled immediately go into plan.md's
     Deferred Actions section
     - Check this section first when starting implementation

     CLAUDE.md Changes

     Add a new bullet under "Before You Start" item 2 (the plugin-specific
     reads), or as a new top-level section linking to the protocol doc.
     Keeps CLAUDE.md lean — just a link.

     Acceptance Criteria

     Given a session that enters plan mode
     When Claude creates a plan
     Then a plan.md file exists under prompt-logs/{YYMMDD}-{title}/
     And it contains a "Success Criteria" section with BDD-style acceptance
     tests
     And it contains a "Deferred Actions" section

     Given a user request during plan mode that cannot be acted on
     immediately
     When Claude records it
     Then it appears as a checklist item in plan.md's Deferred Actions
     section

     Given an ongoing session with user interaction
     When Claude learns something from ping-pong or implementation
     Then it is recorded in lessons.md in the same directory as plan.md
     And each entry has at minimum: what was expected, what actually
     happened, takeaway

     Verification

     - Read the created docs/plan-and-lessons.md and confirm it matches the
     protocol
     - Read CLAUDE.md and confirm the link is present and correct
     - Confirm this session's own prompt-logs/260130-plan-lesson/plan.md and
      lessons.md are created as dogfooding
